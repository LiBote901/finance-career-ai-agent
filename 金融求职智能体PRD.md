## 一、产品概述

### 1.1 产品名称
金融行业求职一条龙服务助手

### 1.2 产品定位
覆盖情报搜集→简历优化→笔试准备→面试模拟全流程的金融求职智能体，是用户的专属私人金融求职教练。

### 1.3 目标用户
- **核心用户**：大三、大四准备求职金融行业的学生
- **目标赛道**：券商/投行、基金、银行、PE/VC、量化等金融细分领域

### 1.4 产品价值主张
从"知道机会"到"拿下机会"的一站式服务，将用户从"焦虑地刷信息"的被动状态转变为"有条不紊推进"的主动状态。

---

## 二、市场背景与用户痛点

### 2.1 行业背景
金融行业求职竞争激烈，信息分散、流程复杂。应届生需要在短时间内完成信息搜集、简历投递、笔试准备、面试冲刺等多个环节，而每个环节都存在明显的痛点和信息差。

### 2.2 用户痛点分析

| 痛点 | 现状描述 | 影响 |
|------|---------|------|
| **信息碎片化** | 需要每天刷几十个公众号、论坛、官网，信息分散且易遗漏 | 错失网申窗口，错过优质机会 |
| **简历缺乏针对性** | 一份简历投所有岗位，不了解不同岗位的筛选偏好 | 通过率低，无法突出匹配度 |
| **笔试资源分散** | 行测、专业题、真题分布在各平台，缺乏系统练习 | 准备效率低，刷题无方向 |
| **面试准备不足** | 缺乏专业模拟，面对专业问题心里没底 | 面试表现不稳定，错失offer |

## 三、产品解决方案

### 3.1 整体架构
智能体融合四重专业身份能力，通过五个协同模块实现全链路覆盖：

```
┌─────────────────────────────────────────────────────────────┐
│                    金融求职一条龙智能体                        │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐    │
│  │ 情报分析师 │  │ 简历顾问  │  │ 笔试助教  │  │ 面试考官  │    │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘    │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ 情报搜集模块  │  │ 简历优化模块  │  │ 笔试准备模块  │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│  ┌─────────────┐  ┌─────────────┐                          │
│  │ 面试模拟模块  │  │全链路看板模块│                          │
│  └─────────────┘  └─────────────┘                          │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 四重身份能力

#### 3.2.1 情报分析师
- 实时监控金融招聘渠道
- 结构化呈现岗位信息
- 智能匹配用户背景

#### 3.2.2 简历顾问
- 熟悉金融各赛道的简历筛选规则
- 提供逐条优化建议
- 展示优秀简历范例

#### 3.2.3 笔试助教
- 整合行测、专业知识、性格测试等题库
- 支持按公司或题型专项练习
- 提供错题本功能

#### 3.2.4 面试考官
- 根据目标岗位生成模拟题（行为面、专业面、技术面）
- 支持文字或语音模拟
- 输出评分与改进指导

### 3.3 五大功能模块

#### 模块一：情报搜集模块

**功能描述**：
抓取目标公司官网、应届生论坛及金融求职公众号信息，进行结构化存储与订阅推送。

**核心功能点**：
| 功能 | 说明 |
|------|------|
| 多渠道爬取 | 覆盖30家目标公司官网、应届生论坛、金融求职公众号 |
| 结构化存储 | 岗位名称、URL、发布日期、截止日期 |
| 增量识别 | 对比上一次爬取结果，新增岗位标"NEW"并推送 |
| 定时更新 | 每4小时自动爬取一次 |

**推送规则**：
- 根据用户选择的目标岗位类型，智能匹配并推送
- 支持用户自定义订阅条件（公司、岗位、地点等）

---

#### 模块二：简历优化模块

**功能描述**：
根据岗位画像匹配简历关键字段，通过智能诊断给出优化建议并展示各岗位优秀简历模板。

**核心功能点**：
| 功能 | 说明 |
|------|------|
| 简历上传解析 | 支持PDF/Word格式，自动提取关键信息 |
| 岗位匹配诊断 | 根据投行/行研/量化等岗位偏好，评分并给出优化建议 |
| 关键词检测 | 基于预设关键词库，检测简历关键词覆盖度 |
| 优秀范例展示 | 提供各岗位优秀简历模板供参考 |

**简历评分机制**：
- **投行岗关键词库**：包含"尽职调查""招股书""财务建模""估值""路演材料""反馈意见回复"等45个词，每出现1个加2分
- **行研岗关键词库**：包含"行业研究""公司深度""盈利预测""产业链分析""调研纪要""数据更新"等40个词，每出现1个加2分
- **量化岗关键词库**：包含"因子挖掘""回测""多因子模型""Python""SQL""风险模型""机器学习"等35个词，每出现1个加2分

**迭代机制**：
- 用户可多次上传修改后的简历
- 达到建议阈值（如70分以上）后，提示"简历已达标"

---

#### 模块三：笔试准备模块

**功能描述**：
整合行测、专业题及性格测试，按公司、题型分类并提供错题本功能。

**核心功能点**：
| 功能 | 说明 |
|------|------|
| 题库分类 | 按题型（行测/专业题/性格测试）、按公司（历年真题） |
| 专项练习 | 支持选择特定公司或题型进行练习 |
| 自动批改 | 选择题比对标准答案；简答题关键词匹配判定 |
| 错题本 | 自动记录错题，支持重复练习 |

**自动批改规则**：
- **选择题**：直接比对标准答案
- **简答题**：用户回答中包含3个以上标准关键词即判定正确

**智能推送**：
- 看板中笔试截止日前7天，自动推送"该公司近3年真题专项练习"

---

#### 模块四：面试模拟模块

**功能描述**：
根据岗位生成高频题库，支持文字或语音模拟并输出评分与参考答案。

**核心功能点**：
| 功能 | 说明 |
|------|------|
| 题库生成 | 根据目标岗位生成行为面、专业面、技术面等模拟题 |
| 模拟方式 | 支持文字对话或语音模拟 |
| 智能评分 | 基于预设评分标准对用户回答进行评分 |
| 参考答案 | 提供标准答案及改进建议 |

**面试评分标准示例**：
| 题目 | 评分规则 |
|------|---------|
| DCF模型步骤 | 答出5步以上：8-10分；3-4步：5-7分；不足3步：0-4分 |
| 行业研究框架 | 答出完整框架：8-10分；部分框架：5-7分；零散回答：0-4分 |

**语音模拟技术**：
- 调用Web Speech API将用户语音转文字
- 将文字与参考答案的语义向量做余弦相似度计算
- 输出相似度百分比作为得分

**智能推送**：
- 看板中面试日期前5天，自动生成该公司及岗位的模拟题
- 要求用户完成至少一次模拟才能标记"面试已准备"

---

#### 模块五：全链路看板模块

**功能描述**：
以可视化方式展示求职进度，自动提醒网申截止、笔试倒计时等关键节点。

**核心功能点**：
| 功能 | 说明 |
|------|------|
| 求职进度可视化 | 以看板形式展示各岗位投递状态 |
| 倒计时提醒 | 网申截止、笔试时间、面试时间倒计时 |
| 投递记录管理 | 支持手动录入或自动提取投递信息 |
| 关键节点提醒 | 自动推送重要节点提醒 |

**看板状态**：
```
待投递 → 已投递 → 笔试中 → 面试中 → 已结束
```

## 四、技术架构（KISS模型）

### 4.1 知识层（Knowledge）

静态知识库，不依赖实时更新：

| 知识库类型 | 内容说明 |
|-----------|---------|
| 投行岗简历关键词库 | 45个关键词，每出现1个加2分 |
| 行研岗简历关键词库 | 40个关键词，每出现1个加2分 |
| 量化岗简历关键词库 | 35个关键词，每出现1个加2分 |
| 各岗位面试评分标准表 | 专业题、行为题等评分细则 |

### 4.2 数据层（Information & Data）

动态数据，实时获取并更新：

| 数据类型 | 更新频率 | 说明 |
|---------|---------|------|
| 招聘信息数据 | 每4小时 | 岗位名称、URL、发布日期、截止日期 |
| 用户行为数据 | 实时 | 简历诊断记录、笔试练习记录、面试模拟记录 |
| 看板动态数据 | 实时 | 投递记录、倒计时等 |

### 4.3 技能层（Skills）

可执行能力，不依赖通用大模型空谈：

| 技能 | 技术实现 |
|------|---------|
| 情报抓取 | Python requests + BeautifulSoup（静态页面）；Selenium（需登录页面） |
| 简历文本解析 | 正则表达式提取教育经历；spaCy中文命名实体识别 |
| 笔试自动批改 | 选择题比对标准答案；简答题关键词匹配 |
| 面试语音模拟 | Web Speech API语音转文字；语义向量余弦相似度计算 |

### 4.4 流程层（SOP）

固定协作流程，用户不可跳过：

```
第一步：目标岗位选择
    ↓
第二步：情报推送 → 点击投递 → 进入看板
    ↓
第三步：简历诊断 → 修改简历 → 再次诊断 → 达到阈值
    ↓
第四步：笔试练习（截止日前7天自动推送）
    ↓
第五步：面试模拟（面试日前5天自动生成）
```

**详细流程说明**：

| 步骤 | 触发条件 | 操作说明 |
|------|---------|---------|
| 第一步 | 用户首次使用 | 强制完成"目标岗位选择"（投行/行研/量化/销售交易，至少选1个） |
| 第二步 | 情报模块推送岗位 | 用户点击"投递"后，岗位自动进入看板，触发简历模块生成初始诊断 |
| 第三步 | 简历诊断后 | 用户根据诊断修改简历，可再次上传重新评分，直到达到建议阈值（如70分以上） |
| 第四步 | 笔试截止日前7天 | 笔试模块自动推送"该公司近3年真题专项练习" |
| 第五步 | 面试日期前5天 | 面试模块自动生成模拟题，用户完成至少一次模拟才能标记"面试已准备" |

## 九、风险与应对

| 风险类型 | 风险描述 | 应对措施 |
|---------|---------|---------|
| 数据风险 | 爬取目标网站反爬机制升级 | 建立多源数据渠道，降低单点依赖；优化爬取策略 |
| 内容风险 | 简历模板、面试题库内容质量不足 | 引入行业专家审核，持续迭代优化 |
| 用户风险 | 用户隐私数据泄露 | 加强数据加密，严格权限控制 |
| 竞争风险 | 类似产品出现 | 快速迭代，深化金融垂直领域专业性 |
| 合规风险 | 爬取数据版权问题 | 仅爬取公开信息，注明来源，必要时获取授权 |
